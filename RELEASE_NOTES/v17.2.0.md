# Fina v17.2.0

## Novidades

- **Conexão Open Finance com Klavi.** Fluxo completo de autenticação (Access Key + Secret Key), criação de link, escolha de instituição e consentimento (autorização no navegador). Como a Klavi entrega os dados de forma assíncrona (webhook/console, sem endereço público para o app receber a chamada de volta), a sincronização final é feita importando o JSON do relatório colado em Configurações > Open Finance.
- **Sincronização automática com Belvo.** A agregação Brasil (Open Finance) da Belvo é síncrona: depois de conectar via My Belvo Portal (aberto no navegador), o Fina passa a sincronizar contas, saldos e lançamentos automaticamente, com o mesmo botão **Sincronizar** já usado pela Pluggy.

## Notas técnicas

- Credenciais da Klavi migraram de uma API key única para o par Access Key/Secret Key (mesmo modelo de client id/secret da Pluggy e da Belvo).
- Novo status de conexão `awaiting_import` para refletir "relatório solicitado, aguardando importação" no card da Klavi.
- Parsers de relatório (Klavi) e de resposta de API (Belvo) em `src/shared/utils.ts`, cobertos por testes unitários; ambos são best-effort porque a documentação pública dos dois provedores não expõe o schema completo dos dados retornados — podem precisar de ajuste fino contra credenciais reais de sandbox.
